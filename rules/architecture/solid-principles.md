# SOLID Principles

- Single Responsibility: one reason to change per module/class
- Open/Closed: add new variants by adding files, not editing switch statements
- Liskov Substitution: subtypes must honor the base type's contract
- Interface Segregation: small, focused interfaces over large general ones
- Dependency Inversion: depend on abstractions, inject implementations
