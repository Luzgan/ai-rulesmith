# Incremental Changes

- Break large tasks into small, shippable increments
- Each increment should compile, pass tests, and be reviewable
- Commit after each meaningful chunk — not one giant commit at the end
- If the approach changes mid-task, commit current progress first
- Smaller changes are easier to review and safer to revert
