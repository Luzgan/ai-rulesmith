# Test-Driven Development

- Red: write a failing test that describes the desired behavior
- Green: write the minimum code to make the test pass
- Refactor: clean up while keeping all tests green
- Start with the simplest case, add complexity through additional tests
- For bug fixes: write a test that reproduces the bug first, then fix it
- Only refactor when all tests are green
