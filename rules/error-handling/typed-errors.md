# Typed Errors

- Use custom error types/classes instead of generic `Error`
- Return structured error responses from APIs (consistent shape)
- Distinguish expected errors (validation) from unexpected errors (crashes)
- Include error codes for programmatic handling
- Type error responses so callers can handle them precisely
